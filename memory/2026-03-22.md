## Deep Research Session (Evening) 🔬
Two parallel research efforts:
1. **CC (Claude Code)** — read full plugin source. Found: `streaming.enabled` defaults to `false`, so `streamPath` never reaches Twilio provider → `canStream` always false → never returns `<Connect><Stream>`.
2. **Web Claude** — researched all external APIs. Found: Status callbacks may hit same URL as main webhook → `CallStatus: "ringing"` triggers `"empty"` before the real in-progress webhook fires. Also major insight: **zero audio conversion needed** in entire pipeline (ElevenLabs outputs `ulaw_8000`, OpenAI Realtime API accepts `audio/pcmu` natively).

### Architecture Insight: Zero-Conversion Pipeline
```
Caller mic → PSTN → Twilio (mulaw) → WebSocket (mulaw base64) 
  → OpenAI Realtime STT (accepts pcmu natively) → transcript
  → Claude/LLM → text response  
  → ElevenLabs (outputs ulaw_8000) → base64 pipe directly to Twilio WS
  → Twilio → PSTN → Caller ear
```
**Target latency: 725ms–1.25s voice-to-voice.** Genuinely conversational.
